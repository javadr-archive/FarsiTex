Yes, you're right. I took a look at the source code and found 
out that I've forgotten to handle this case; however, I must 
mention that the lines in the editor cannont grow larger than 
256 characters in any case. The best thing I can do is to 
truncate the line or wordwrap it. Getting rid of this problem 
doesn't require any effort as I already have the code for 
cut/paste word-wrapping.

Afterall, this is not a big deal. Although the editor is supposed 
to open plain ascii files and automatically convert them to ftex 
format, nobody is using this feature anymore because copying 
ascii text from windows clipboard is a far better solution!
